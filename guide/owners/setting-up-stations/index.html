    Setting Up Stations¶

    Most Feudalism objects are configured through their Object Description (in the Edit window). Type the settings as comma-separated values. This page lists the description format for each common station type.

    Security is almost universal

    Nearly every object accepts a security token — owner, group, or all — controlling who may use it. Many also accept debug to print extra info to you while you set them up.

    Gatherables (bushes, fruit, wood, generic nodes)¶

    <itemSlug>,<qty>[,<itemSlug>,<qty>...] ,<maxClicks>,<reset|die>,<resetSeconds>,<cooldownSeconds>[,security][,debug]

    One click always grants every listed item (predefined amounts). Repeat item,qty pairs before the trail. The trail starts at the first field that is a number whose next field is reset or die (after the last qty you put maxClicks, then the action).

    Field Notes
    reset / die Reset = reappears after the timer; die = destroyed when depleted.
    cooldown 1.0–30.0 seconds between clicks.
    reset time 1.0–3600.0 seconds.

    Examples

    banana,1,7,reset,300.0,3.0,all
iron ore,2,5,reset,600.0,2.0,group
rare herb,1,1,die,60,5.0,owner
log,2,5,reset,600.0,3.0,all
mulberry leaf,10,mulberry,10,6,reset,300,2,all

    Range: 10 m. Coins (gold coin, etc.) cannot be gathered.

    Mining node (texture-changing)¶

    <itemId>,<qtyPerClick>,<maxClicks>,<reset|die>,<resetSeconds>,<cooldown>[,security][,debug][,activeTexture][,depletedTexture]

    iron ore,1,10,reset,600,3,all
copper ore,1,8,reset,900,4,group,,copper,rock

    Mine (multi-deposit)¶

    • Description: mine,gather,gathering,all
    • Notecard named deposits (see the Recipe Notecard Reference).
    • Range: 15 m.

    Water well¶

    <security>[,debug]

    Touch within 5 m → Get Water fills a worn skin or bucket with water.

    Huntable animal¶

    <animalName>,<health>,<resetDelaySec>,<meleeCR>,<security>[,debug]

    deer,100,30,2,all
boar,150,60,3,group
wolf,80,20,4,all,debug
    • health — hit points; reset delay — respawn seconds (min 5).
    • melee CR — melee defense, bow difficulty base, and melee kill XP.
    • Loot: every object inside the animal drops on death.
    • Spawn points (owner): long-touch (~0.8 s) at up to 40 locations.

    Oven / cauldron / fermenter¶

    Touch mode:

    <noun>,<verb>,<verb-ing>,<ingredient-word>,<recipe-word>[,security][,fire]

    oven,cook,cooking,ingredients,recipe,all

    Sit mode — prefix with sit,:

    sit,oven,cook,cooking,ingredients,recipe,all

    Add fire to require a lit fire. Recipes go in a recipes notecard.

    Scripts (four per object): F4 Crafting - Main, Recipes, Menu, Delivery.
    Do not use legacy Oven Script on new stations. See DropBox & physical delivery for central physical inventory.

    Food plate¶

    <itemId>,<displayName>,<healthMod>,<staminaMod>,<maxServings>,<security>[,debug]

    stew,Hearty Stew,5,3,12,all
bread,2,2,8,group

    (The short form omits the display name.) Range: 5 m.

    Producer (press, churn, malting tub)¶

    <mode>,<noun>,<verb>,<verb-ing>[,security][,debug]
    • mode = static (touch) or animated (AVsitter sit).
    • Recipes go in a recipes notecard (producer format).
    static,press,press,pressing,all
animated,churn,churn,churning,group

    Farm field / herb pot / grape vine¶

    • Description: <DisplayName>,<security> — e.g. Corn,all or Parsley,all.
    • Notecard named config (one recipe line). See the reference.
    • Range: 15 m; up to 9 plant prims named 0–8.

    Grain mill / grain silo¶

    • Description: <DisplayName>,<security> — e.g. Grain Mill,all.
    • Notecard named config.

    Compost bin¶

    <outputItem>,<security>

    fertilizer,all

    Accepts: stover, scraps, straw, manure, slop, eggshells. Range: 5 m.

    Renewable harvest / livestock¶

    Renewable harvest:

    <noun>,<verb>,<verb-ing>,<itemId>,<maxAmount>,<unitsPerTick>,<tickSeconds>[,security][,debug]

    sheep,shear,shearing,wool,20,1,120,all

    Livestock (huntable + harvestable) adds combat fields:

    sheep,80,60,2,wool,20,1,120,shear,shearing,all

    Refillable container (barrel, keg, milk pail)¶

    <itemName>,<security>[,maxQty][,debug]

    milk,all
wine,all,100
wine*,all,500

    Liquids use the * suffix and a drink skin.

    Storage chest¶

    <security>[,debug]

    owner
group
all,debug

    Sharpening wheel¶

    sharpening wheel,sharpen,sharpening,<security>

    Smith / skinning / multi-output workstation¶

    Object description (touch mode):

    <noun>,<verb>,<verb-ing>,<parts-word>,<recipe-word>[,security][,<craftSeconds>]

    Example:

    skinning station,skin,skinning,parts,recipes,all

    Scripts (four per object): F4 Crafting - MultiItem - Main, Recipes, Menu, Delivery.
    Recipes: recipes notecard — see Recipe Reference (multi-output section).

    DropBox (central physical inventory)¶

    Dedicated object for physical crafting outputs used by stations that do not hold those objects locally.

    • Install F4 DropBox.lsl + F4 DropBox - Admin.lsl
    • Next Owner Copy: NO
    • Optional dropbox.cfg notecard
    • Accept Experience; allow parcel HTTP-in

    Full setup: DropBox & physical delivery.

    Archery target / dummy¶

    Practice target:

    <name>,<baseCR>,<ringCount>,<marginPerRing>,<security>,<roleplay|arcade>

    Practice Butt,3,5,10,all,roleplay

    Practice dummy:

    <name>,<health>,<resetDelay>,<meleeCR>,<security>

    Practice Dummy,100,60,4,all

    Managed ranges use a controller plus numbered firing pads — see your range builder guide.

    Always reset after editing

    Description changes usually apply on edit, but notecard changes require a script reset or re-rez.
